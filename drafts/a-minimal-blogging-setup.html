A minimal blogging setup

Sat 17 October 2020
By Duong Nguyen
Category: Misc
Tags: blog pelican github-pages

Motivation

As a lifelong learner, I often want to write and share about what I've learned. I've been looking for a minimal (and hopefully free) blogging setup, which isn't time-consuming to configure, while still flexible enough for customization. There are a ton of full-fledged options out there, such as Medium, Tumblr, Wordpress etc., but I've been leaning towards more lightweight/casual options such as static-site generators (e.g. Hugo, Pelican just to name a few). I was also impressed with the simplicity of a recently introduced fast.ai template. As an attempt to revive my dusty Pelican-powered blog, I decide to give Pelican another try due to its relatively simple setup and growing plugins. And it's written in Python so easier for me to grok. Since I've found most tutorials on this topic outdated, this post will serve as an updated reference to my future self and interested readers. I plan to keep it a living post!

Steps

  1. Start with the official Pelican documentation
  2. Set up pyenv and virtualenv

  3. Install minimally required packages (alternatively, run pip install -r requirements.txt)

    • pip install pelican Markdown
    • pip install ghp-import: easy to publish to Github pages. More on this later.
    • pip install jupyter ipython nbconvert matplotlib: for working with Jupyter notebook
    • pip install beautifulsoup4: needed for TOC generation?
    • pip install pelican-render-math: rendering math/latex
    • pip install pelican-jupyter: for notebook markup

  4. Run pelican-quickstart command to generate proper folder structure and necessary files, including:

    • pelicanconf.py
    • publishconf.py
    • Makefile, tasks.py

  5. Configure plugins

    • Modify pelicanconf.py
    • See doc for more details.

  6. Useful commands

    • Themes:
      • pelican-themes -l: List all installed themes
      • pelican-themes --install /path/to/theme --verbose: install a theme
        • Specify THEME=<theme-name> in pelicanconf.py
        • See doc for more details about themes.
    • Build:
      • make clean: Clean up generated files (e.g., in output/ folder)
      • make html: Generate html files in output/ from content files in content/ folder
      • make serve: Start a local server (at localhost:8000)
      • make devserver: Serve and regenerate files on the fly with content update
    • Publish (free hosting at Github Pages):
      • make publish: Generate files ready for publish
      • make github: Copy published files into Github Pages-specific branch and push to the remote master branch in the username.github.io repo.
        • Note we use a different branch for published content and don't push local's master branch (containing blog configuration and content source) to the remote master
      • See the Makefile for more details. It's pretty readable!

  7. Optional

    • For content that doesn't change very often and not dated in nature, such as About or Contact pages, put them under content/pages/ folder
    • Set up comment channels, such as Disqus.
    • Find/create/customize themes (currently use simple theme as I'm a minimalist!)
    • Configure TOC autogen (somehow extract_toc plugin doesn't work for me)

  8. Main places for custom config

    • pelicanconf.py
    • publishconf.py
    • Makefile
Proudly powered by Pelican, which takes great advantage of Python.
